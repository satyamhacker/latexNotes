Introduction to React Native ### Title / Short Summary React Native ek framework hai jo ek hi code se iOS aur Android apps banata hai. ### Kya hai? (What?) React Native ek JavaScript-based framework hai jo native mobile apps banane ke liye use hota hai. ### Kyu important hai? (Why?) Ek hi codebase se dono platforms ke liye apps bana sakte ho, jo time aur effort bachata hai. ### Kab/use karna chahiye? (When?) - Jab cross-platform app banana ho with single codebase. - **Alternative:** Native development (Swift for iOS, Kotlin for Android), lekin zyada effort lagega. ### Agar use nahi kiya to kya hoga? (If not used / consequences) - Alag-alag platforms ke liye separate teams aur code chahiye, jo costly hai. - Development time zyada lagega. ### Step-by-step explanation / Concept breakdown - **Step 1:** React Native JavaScript aur React ke concepts use karta hai. - **Step 2:** Ek codebase se iOS aur Android dono ke liye apps banati hai. - **Step 3:** Native components (jaise buttons, text) directly use hote hain. - **Step 4:** Hot reloading se code changes real-time dikhte hain. - **Step 5:** Third-party libraries se features jaldi add ho jate hain. ### Code example ```javascript import React from 'react'; import { Text, View } from 'react-native'; const App = () => { return ( Hello, React Native! ); }; export default App; ``` **Line-by-line explanation:** - `import React from 'react'`: React library import karta hai for JSX. - `import { Text, View } from 'react-native'`: Native components import karta hai. - `const App = () =>`: Functional component define karta hai. - ``: Screen ko center karta hai. - `Hello, React Native!`: Text display karta hai. - `export default App`: Component ko export karta hai taaki app me use ho sake. **Quick run expected output:** - Screen pe "Hello, React Native!" center mein dikhega. ### Common beginner mistakes - React aur React Native ke components confuse karna. - Basic JavaScript ya React concepts na samajhna. - Project setup ke time dependencies check na karna. ### Real-world example / use-case - **Scenario 1:** Ek startup ke liye ek to-do app dono platforms pe jaldi banani hai. - **Scenario 2:** Ek e-commerce app me products list dono iOS aur Android pe dikhani hai. ### Quick checklist / TL;DR - React Native cross-platform apps banata hai. - JavaScript aur React knowledge chahiye. - Hot reloading se development fast hota hai. - Third-party libraries features add karti hain. ### FAQs 1. **Q: React Native kab use karu?** A: Jab ek hi code se iOS aur Android apps banani ho. 2. **Q: Isko optimize kaise karu?** A: Clean code likho aur latest React Native version use karo. 3. **Q: Koi alternative hai?** A: Native development (Swift/Kotlin) ya Flutter, lekin zyada effort lagega. 4. **Q: React Native seekhna mushkil hai?** A: Agar JavaScript aur React aata hai, toh asaan hai. 5. **Q: Hot reloading kya hai?** A: Code change karte hi app me changes real-time dikhte hain. ### Practice exercise 1. **Task:** Ek simple app banao jo "Hello, React Native!" dikhaye. **Expected result:** Text screen ke center mein dikhe. 2. **Task:** App me ek aur Text component add karo with different message. **Expected result:** Do text messages ek ke niche ek dikhein. ### Further reading / commands / links - (When?) - Jab React Native app development shuru karna ho. - **Alternative:** Expo CLI, jo setup ko simpler banata hai. ### Agar use nahi kiya to kya hoga? "AwesomeApp" folder banega. - Emulator pe "Welcome to React Native" screen dikhegi. ### Common beginner mistakes - Purana CLI command (`npm install -g react-native-cli`) use karna. - Emulator ya device ready na rakhna. - Dependencies install na karna. ### Real-world example / use-case - **Scenario 1:** Ek to-do app ke liye project setup karo. - **Scenario 2:** Ek prototype app jaldi test karna hai. ### Quick checklist / TL;DR - `npx react-native@latest init` use karo. - Emulator ya device ready rakho. - `run-android` ya `run-ios` se test karo. ### FAQs 1. **Q: Project setup kab karu?** A: Jab naya app shuru karna ho. 2. **Q: Isko optimize kaise karu?** A: Latest CLI version use karo aur clean setup rakho. 3. **Q: Koi alternative hai?** A: Expo CLI simpler hai, lekin native features ke liye React Native CLI better hai. 4. **Q: App run nahi ho rahi?** A: Emulator, dependencies, aur `ANDROID_HOME` check karo. 5. **Q: Default screen kya dikhata hai?** A: "Welcome to React Native" message dikhta hai. ### Practice exercise 1. **Task:** `npx react-native@latest init MyApp` se project banao. **Expected result:** Welcome screen emulator pe dikhe. 2. **Task:** `App.js` kholo aur default text badal kar "My First App" karo. **Expected result:** Screen pe naya text dikhega. ### Further reading / commands / links - **Step 2:** StyleSheet.create use karke styles define karo. - **Step 3:** External StyleSheet file bana sakte ho for large projects. - **Step 4:** Flexbox use karke layouts banao. - **Step 5:** Platform-specific styles ke liye `Platform` module use karo. ### Code example ```javascript import React from 'react'; import { StyleSheet, Text, View } from 'react-native'; const App = () => { return ( Hello, React Native! ); }; const styles = StyleSheet.create({ container: { flex: 1, justifyContent: 'center', alignItems: 'center', }, text: { fontSize: 16, color: 'blue', }, }); export default App; ``` **Line-by-line explanation:** - `container: { flex: 1, justifyContent: 'center', alignItems: 'center' }`: View ko full screen aur centered rakhta hai. - `text: { fontSize: 16, color: 'blue' }`: Text ka size 16 aur color blue set karta hai. **Quick run expected output:** - Screen pe "Hello, React Native!" blue color mein center mein dikhega. ### Common beginner mistakes - Inline styles ka zyada use karna (performance slow karta hai). - Flexbox properties (justifyContent, alignItems) na samajhna. - StyleSheet import bhoolna. ### Real-world example / use-case - **Scenario 1:** Ek login screen me button ko bold aur colorful karo. - **Scenario 2:** Ek list app me text ko centered aur readable banao. ### Quick checklist / TL;DR -
